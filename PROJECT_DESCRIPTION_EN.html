PROJECT: ERP SYSTEM
SALES MANAGEMENT FOR ELECTRONICS MANUFACTURING PLANT

I. INTRODUCTION

1.1. Project Title

Building an ERP (Enterprise Resource Planning) system to manage the Sales process of an electronics manufacturing plant.

1.2. Objectives

  • Automate the sales process from quotation to delivery
  • Efficiently manage customers and transaction history
  • Track sales performance in real-time
  • Provide sales data reporting and analysis

1.3. Scope

The system includes: Customer Management (CRM), Product Management, Quotation Management, Order Management, Delivery Management, Invoice and Payment Management, Reports and Statistics.

II. TECHNOLOGY STACK

2.1. Frontend

  • Framework: React 18 + Vite
  • UI Library: Ant Design
  • State Management: Redux Toolkit
  • Form & Validation: React Hook Form + Yup

2.2. Backend

  • Runtime: Node.js 18+
  • Framework: Express.js
  • Database: MongoDB + Mongoose
  • Authentication: JWT (JSON Web Token)

2.3. Deployment

  • Frontend: Vercel
  • Backend: Railway
  • Database: MongoDB Atlas

III. USER ROLES & PERMISSIONS

No. Role Main Permissions
1 Admin Full system administration, user management, configuration
2 Sales Manager Manage sales team, approve orders, view reports
3 Sales Staff Create quotations, orders, manage customers
4 Customer Service Handle complaints, update order status
5 Warehouse Staff Export goods, update inventory
6 Accountant Issue invoices, track receivables
7 Delivery Staff Update delivery status
8 Director View all reports (read-only)

IV. MAIN FEATURES

4.1. Customer Management (CRM)

  • Create, edit, delete customer information
  • Classify customers (VIP, regular, new)
  • Transaction and interaction history
  • Assign customers to sales staff

4.2. Product Management

  • Electronics product catalog
  • Inventory management and alerts
  • Detailed information and product images

4.3. Quotation and Order Management

  • Create quotations with discounts
  • Convert quotations to orders
  • Track order status
  • Order approval

4.4. Delivery Management

  • Create delivery notes
  • Assign delivery staff
  • Track delivery status

4.5. Invoice and Payment Management

  • Create and issue invoices
  • Track receivables
  • Record payments

4.6. Reports and Statistics

  • Overview dashboard
  • Sales reports by multiple criteria
  • Employee performance reports
  • Charts and visualization

V. USE CASE DIAGRAM

5.1. Actors

The system has 8 main actors:

  • Admin: System administrator
  • Sales Manager: Sales team manager
  • Sales Staff: Sales representative
  • Customer Service: Customer support staff
  • Warehouse Staff: Warehouse employee
  • Accountant: Accounting staff
  • Delivery Staff: Delivery personnel
  • Director: Company director

5.2. Main Use Cases

Function Group Use Cases
User Management Login, Manage Accounts, Assign Permissions
Customer Management View Customer List, Create Customer, Update Info, View History
Product Management View Product List, Manage Inventory, Low Stock Alert
Quotation Management Create Quotation, Approve Quotation, Convert to Order
Order Management Create Order, View Orders, Approve Order, Update Status
Delivery Management Create Delivery Note, Assign Delivery, Update Status
Invoice Management Create Invoice, Issue Invoice, Track Receivables, Record Payment
Reports & Statistics Dashboard, Sales Report, Performance Report

5.3. Main Business Flows

Flow 1: From Quotation to Order

Sales Staff creates quotation → Sales Manager approves (if needed) → Convert to order → Sales Manager approves order (if high value)

Flow 2: From Order to Delivery

Order approved → Warehouse Staff creates delivery note → Delivery Staff receives assignment → Updates status → Confirms successful delivery

Flow 3: From Delivery to Payment

Successful delivery → Accountant creates invoice → Issues invoice → Tracks receivables → Records payment

VI. SYSTEM ARCHITECTURE

6.1. 3-Tier Model

  • Presentation Layer: React + Vite (Frontend)
  • Application Layer: Node.js + Express (Backend API)
  • Data Layer: MongoDB (Database)

6.2. Database Collections

  • users: User information and authentication
  • customers: Customer information
  • products: Product catalog
  • quotations: Quotations
  • orders: Orders
  • deliveries: Delivery information
  • invoices: Invoices and payments

6.3. Security

  • Login with username and password
  • Password encryption
  • Role-based permissions

VII. BENEFITS

7.1. For Business

  • Process automation, reduced processing time
  • Centralized customer management
  • Track sales team performance
  • Reduce errors in quotations and orders
  • Reporting and analysis for decision making

7.2. For Employees

  • User-friendly interface, easy to use
  • Access anytime, anywhere
  • Reduce manual work
  • Track personal KPIs

VIII. CONCLUSION

The ERP Sales Management System is a comprehensive solution that helps automate and optimize the sales process for electronics manufacturing plants. With modern technology (React, Node.js, MongoDB) and flexible architecture, the system ensures:

  • Efficiency: Process automation, reduced processing time
  • Accuracy: Reduced errors in quotations and orders
  • Transparency: Track the entire process
  • Security: Clear permissions, data protection
  • Scalability: Easy to add new features

The project can be deployed on cloud (Vercel + Railway + MongoDB Atlas) with reasonable costs.
